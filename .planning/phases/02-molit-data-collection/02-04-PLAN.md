**1. collect_district(conn, client, molit, lawd_cd, months, deal_type) -> int**

Follow RESEARCH Pattern 5 exactly:
```python
import asyncio
import sqlite3
import httpx
from loguru import logger
from pipeline.clients.molit import MolitClient
from pipeline.config.regions import PIPELINE_REGIONS
from pipeline.utils.idempotency import is_collected, mark_collected
from pipeline.processors.normalizer import normalize_trade_item, normalize_rent_item, aggregate_monthly, get_month_range
from pipeline.storage.repository import upsert_apartment, insert_monthly_prices

async def collect_district(
    conn: sqlite3.Connection,
    client: httpx.AsyncClient,
    molit: MolitClient,
    lawd_cd: str,
    months: list[str],
    deal_type: str,
) -> int:
    """Collect all months for one district and deal_type. Returns total rows inserted."""
    total = 0
    for deal_ym in months:
        if is_collected(conn, lawd_cd, deal_ym, deal_type):
            continue
        items = await molit.fetch_all(client, lawd_cd, deal_ym, deal_type)
        if deal_type == "trade":
            normalized = [normalize_trade_item(i) for i in items]
        else:
            normalized = [normalize_rent_item(i) for i in items]
        aggregated = aggregate_monthly(normalized, deal_type)
        for agg_row in aggregated:
            apt_id = upsert_apartment(
                conn, lawd_cd,
                agg_row["apt_nm"], agg_row["umd_nm"],
                jibun=agg_row.get("jibun"),
                road_nm=agg_row.get("road_nm"),
                build_year=agg_row.get("build_year"),
            )
            total += insert_monthly_prices(conn, apt_id, [agg_row])
        mark_collected(conn, lawd_cd, deal_ym, deal_type, record_count=len(aggregated))
        logger.debug(f"Collected {lawd_cd} {deal_ym} {deal_type}: {len(aggregated)} agg rows")
    return total
```

CRITICAL: Do NOT call conn.commit() inside the inner loop (per-row). The commit happens in insert_monthly_prices (after the batch) and mark_collected. This is correct.

CRITICAL: When aggregate_monthly groups normalized items, the agg_row has keys apt_nm, umd_nm, deal_ym. But the normalize functions may not produce jibun/road_nm in the aggregated output. Collect these from the first raw normalized item in the group. Simplest fix: pass jibun=None, road_nm=None to upsert_apartment — the apartment was already inserted from trade data and these are informational only.

**2. collect_all_regions(conn, molit, start_ym="200601") -> dict**

Synchronous entry point that creates an httpx.AsyncClient and runs collect_district for all 29 regions × both deal_types:

```python
def collect_all_regions(
    conn: sqlite3.Connection,
    molit: MolitClient,
    start_ym: str = "200601",
) -> dict:
    """
    Run full collection for all PIPELINE_REGIONS, both trade and rent.
    Returns summary dict: {"trade": N, "rent": N, "regions": 29}.
    """
    months = get_month_range(start_ym)
    summary = {"trade": 0, "rent": 0}

    async def _run():
        async with httpx.AsyncClient() as client:
            for name, lawd_cd in PIPELINE_REGIONS.items():
                logger.info(f"Collecting {name} ({lawd_cd}) — {len(months)} months × 2 types")
                for deal_type in ("trade", "rent"):
                    n = await collect_district(conn, client, molit, lawd_cd, months, deal_type)
                    summary[deal_type] += n

    asyncio.run(_run())
    summary["regions"] = len(PIPELINE_REGIONS)
    return summary
```

NOTE: asyncio.run() creates a new event loop — do NOT call from inside an already-running async context. This is the correct pattern for a sync CLI entry point.
