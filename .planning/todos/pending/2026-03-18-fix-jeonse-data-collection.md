## Problem

전세가(전세 거래) 데이터가 수집되지 않는 버그. MOLIT API에서 전세 거래 유형을 누락하거나 필터링 로직에 문제가 있을 가능성.

- MOLIT API는 매매(매매가)와 전세(전세가) 거래를 별도 엔드포인트 또는 파라미터로 구분
- 현재 파이프라인이 매매 거래만 수집하고 전세 거래는 누락하는 것으로 보임
- 수집 후 DB에 전세가 레코드가 없거나 0건인 상태

## Solution

1. MOLIT API 엔드포인트 확인 — 전세 거래용 API (`아파트전월세`) 별도 호출 필요한지 확인
2. 컬렉터 로직에서 거래 유형 파라미터(`dealType` 또는 유사) 확인
3. DB 스키마에 전세가 필드(`jeonse_price`) 존재 여부 확인
4. 누락된 경우 전세 거래 수집 로직 추가
